| F-01 | Medium | Local | **Δ4's oracle is not implementable with the harness it is bound to.** Δ4 places P-SCOPE-DEG-01 in `hookCompatibility.test.js` under `degradedEnv()` and asserts exit 0. But that file's `runHookScript` spawns `spawnSync("bash", …)` **by name** with the child-env merge (`hookCompatibility.test.js:48–52`), and per verified claim C17 (TE TSPEC F-02), spawn resolution uses the *child* env's `PATH` — under `degradedEnv()`'s empty `PATH` the spawn itself fails `ENOENT` (`status: null` → harness exitCode −1), so the test fails before the script runs, regardless of the script's behavior. Fix: Δ4 must specify the spawn mechanism — e.g. route through `runScopeCheck` (PLAN TASK-02) with the `BASH_ABS` absolute-path spawn, or state that this one case uses `spawnSync(BASH_ABS, [scriptPath], …)`. | Δ4; PROP-SCOPE-02 oracle |
