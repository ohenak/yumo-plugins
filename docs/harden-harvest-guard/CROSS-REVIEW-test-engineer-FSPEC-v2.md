## Positive Observations

- **Every one of the nine iteration-1 dispositions survives adversarial re-derivation** — not one is a paper resolution. In each case I attempted to construct an implementation that passes the new row while keeping the old defect, and failed: M67 kills first-char-digit, M68/M70 kill word-verbs-only, M71 kills parse-first, M73 kills unwrap-once, M74 kills D3-first `mv`, M79 kills two-operand `mv`, S07 kills the missing basename filter.
- The M77 resolution is *stronger* than requested: the deterministic `NOT_COMMITTED` choice (SE F-06) is what makes the row a sufficient oracle at all — with a TSPEC-chosen "nearest G-state" the row would have had two defensible expected codes. Good example of an SE and a TE finding resolving each other.
- Matrix ownership was handled exactly right: rows promoted to the REQ (single oracle, no FSPEC-local shadow matrix), with the M33 re-run set and its exclusion rationale updated in the same commit — and the exclusion rationale is itself correct, not boilerplate.
- The over-cap recursion resolution (opaque → D1 ALLOW, RR-3) is the only reading that keeps both NFR-01 and the cap testable; treating over-cap as indeterminate would have made M-row outcomes depend on an unpinned TSPEC constant.
- The two residual findings are Lows precisely because the v1.1 design keeps them cheap: each is one added matrix row with an already-constructible fixture, not a flow change.
